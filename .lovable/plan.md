

## Plan: Admin TV Devices Dashboard & Audit Bug Fix

### 1. Fix Audit Log Bug in All Three Edge Functions

The `tv_pairing_audit_logs` table requires `organization_id NOT NULL`. Three edge functions are inserting without it:

**`tv-pairing-validate/index.ts`**: The device is already fetched with `select('*, properties(id, title)')`. The `device` object has `organization_id` from the `tv_device_pairings` table. Add `organization_id: device.organization_id` to the audit insert.

**`tv-device-register/index.ts`**: The `property.organization_id` is already available. Add it to the audit insert.

**`tv-device-unpair/index.ts`**: The device select only fetches `id, guest_email`. Expand it to also select `organization_id`, then include it in the audit insert.

### 2. Build TVDevicesSettingsPage.tsx

Replace the placeholder with a full admin dashboard:

- Import `useAdminOrganizationId` and `useTVDeviceAdmin`
- **Stat cards row**: Total Devices, Online (green), Offline (red) — derived from `deviceCounts`
- **Device table** with columns: Device Name, Property, Status (online/offline badge based on `isDeviceOnline`), Display Mode, Actions

### 3. Add "Pair New TV" Modal

A dialog/modal triggered by a button at the top of the page:

- **Property dropdown**: Fetch org properties via `supabase.from('properties').select('id, title').eq('organization_id', orgId)`
- **Pairing code input**: 6-digit text input
- **Device name input**: Free text (e.g., "Living Room TV")
- **Submit**: Calls `supabase.functions.invoke('tv-device-register', { body: { property_id, device_id: crypto.randomUUID(), device_name } })` — this creates the device record. Then validates the pairing code via `tv-pairing-validate` with a placeholder admin email. Actually, since this is a host-initiated flow, the register function already creates the device with a NEW pairing code. The admin flow should instead: look up the device by the code the TV is showing, then pair it directly.

**Revised approach**: The TV is already showing a pairing code (generated by `tv-device-register` which the TV app calls on boot). The admin enters that code. We call `tv-pairing-validate` with `pairing_code` and an admin identifier (e.g., `host@admin`) to mark it paired. We also update `device_name` separately. This keeps the existing flow intact.

Implementation:
1. Admin enters the 6-digit code showing on the TV + selects property + names device
2. Call `tv-pairing-validate` with `{ pairing_code, email: 'host-paired' }` to pair
3. Then update the device name via direct Supabase update: `supabase.from('tv_device_pairings').update({ device_name }).eq('pairing_code', ...)` — but the code is cleared after pairing. So instead, use the returned `device_id` to update the name.
4. Refetch device list
